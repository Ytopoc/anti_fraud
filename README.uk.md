# Виявлення шахрайських користувачів - SKELAR x mono AI

ML-пайплайн для змагання SKELAR x mono AI, що передбачає, чи є користувач
шахраєм (`is_fraud ∈ {0, 1}`), на основі метаданих реєстрації та історії
транзакцій. Пайплайн будує великий набір поведінкових ознак для кожного
користувача і навчає ансамбль із 3 моделей gradient boosting
(LightGBM + CatBoost + XGBoost) з OOF-блендингом і підбором порогу,
оптимізованим за F1.

## Структура проєкту

```
Anti_fraud/
├── data/
│   ├── train_users.csv          # 395k користувачів з міткою is_fraud
│   ├── test_users.csv           # 169k користувачів для прогнозу
│   ├── train_transactions.csv   # 3.1M транзакцій
│   └── test_transactions.csv    # 1.35M транзакцій
├── features_v9.py               # Інженерія ознак (~250 ознак на користувача)
├── ensemble_v9.py               # Повний пайплайн + опційне порівняння v8 vs v9
├── submission.csv               # Вихід: id_user, is_fraud
├── sub.csv                      # Снапшот попереднього сабмішну
├── catboost_info/               # Артефакти тренування CatBoost
└── README.md
```

## Схема даних

**Users** (`*_users.csv`):
`id_user, timestamp_reg, email, gender, reg_country, traffic_type, is_fraud`

**Transactions** (`*_transactions.csv`):
`id_user, timestamp_tr, amount, status, transaction_type, error_group,
currency, card_brand, card_type, card_country, card_holder, card_mask_hash,
payment_country`

Доля шахрайства у train ~4–5%, тому пайплайн використовує
`scale_pos_weight = N_neg / N_pos` для балансу класів замість ресемплінгу.

## Як запустити

Пайплайн самодостатній і визначає шляхи відносно власного розташування.
З кореня проєкту:

```bash
python ensemble_v9.py
```

`ensemble_v9.py` запускає v9-пайплайн і **якщо у тій самій папці присутній
`features_v8.py`** - додатково тренує v8-ансамбль для чесного порівняння
поряд. Якщо `features_v8.py` відсутній, v8-блок автоматично пропускається,
і для фінального сабмішну використовується v9.

Вихід записується у `submission.csv` поряд зі скриптом (`id_user, is_fraud`).

## Інженерія ознак (`features_v9.py`)

Приблизно 250 ознак на користувача, згрупованих у:

1. **Агрегати транзакцій** - кількості, статистика сум (mean/max/min/std/
   median/skew/IQR), статистика часу від реєстрації, лічильники типів
   транзакцій, лічильники груп помилок, прапорці неспівпадіння країн
   (карта vs реєстрація vs платіж), нічна активність, довжина імені
   власника карти.
2. **Серії та зміни статусів** - максимум підряд невдач, патерн "невдача -
   зміна карти", невдачі до першого успіху.
3. **Динаміка по карті / по власнику** - тривалість життя карти, частка
   невдач на карту, карти з повністю невдалою історією, власники зі
   спільними картами, сесії, розділені паузами 30+ хв.
4. **Вікна ранньої активності** - кількості/невдачі/карти/власники в межах
   1 / 6 / 24 / 72 / 168 годин після реєстрації.
5. **Збіг email-власник** - евристична схожість між локальною частиною
   email і домінантним іменем власника карти.
6. **Поведінковий зсув** - частка невдач у першій vs другій половині,
   різноманітність карт.
7. **v9 крос-користувацькі сигнали**
   - Спільне використання карт між користувачами (карта, що
     використовується 2+ користувачами - сильний fraud-сигнал).
   - Спільні імена власників між користувачами.
   - **Токсичність карти** - leave-one-out target encoding того, наскільки
     шахрайськими є інші користувачі тієї ж карти.
   - **Токсичність власника** - те саме, але для імен власників.
8. **v9 target encoding на рівні транзакцій** - OOF target encoding для
   `card_country`, `card_brand`, `currency`, `payment_country`, агреговані
   на користувача mean і max (5-fold, smoothing = 50).
9. **v9 спеціалізовані ознаки low-card** - неспівпадіння країни для
   єдиної карти, єдина карта з помилками antifraud/fraud, init-only
   користувачі, неактивні користувачі з невдалим `card_init`.
10. **Прискорення темпу** - перша похідна часових проміжків між
    транзакціями; шахраї схильні прискорюватись, знаходячи робочі карти.
11. **Target encoding на рівні користувача** - для `reg_country`,
    `traffic_type`, `email_domain`, `gender × traffic`, `country × traffic`.

Усі target encoding використовують 5-fold OOF на тренувальній вибірці і
повну статистику тренування на тестовій, тому витоку міток немає.

## Ансамбль моделей (`ensemble_v9.py`)

5-fold `StratifiedKFold` (seed 42) з трьома різноманітними бустерами:

| Модель       | Форма дерева                     | Learning rate | Примітки                                |
| ------------ | -------------------------------- | ------------- | --------------------------------------- |
| LightGBM     | 255 листя, depth = -1            | 0.02          | Агресивний, низький subsampling ознак   |
| CatBoost     | depth 8                          | 0.03          | Власна обробка high-cardinality TE      |
| XGBoost      | 255 листя, `lossguide`           | 0.02          | Histogram-based                         |

Усі три використовують `scale_pos_weight` для дисбалансу класів і 300-раундів
early stopping за AUC.

### Блендинг

Чотири методи блендингу оцінюються на OOF-передбаченнях; перемагає той,
що дає найкраще F1:

- Зважене середнє (grid по вагах, що сумуються в 1.0).
- Rank-based блендинг (той самий grid, але на rank-percentile кожної моделі).
- Stacking логістичною регресією (level-2 LR на сирих OOF).
- Поліноміальний stacking (degree-2 LR з взаємодіями).

Поріг підбирається перебором `0.05 → 0.95` (крок 0.005) на максимум F1
по фінальних OOF-ймовірностях.

## Що пайплайн виводить

Окрім запису сабмішну, прогін друкує:

- F1, precision, recall, PR-AUC, Brier score - по моделях і блендованим OOF.
- Top-k precision/recall на 1 / 3 / 5 / 10 % користувачів.
- Confusion matrix і повний classification report за обраним порогом.
- Таблицю розгортки порогу від 0.10 до 0.95.
- **Ablation-аналіз** - падіння F1 при видаленні кожного v9-блоку ознак
  (cross-user card, card toxicity, holder toxicity, cross-user holder,
  tx-level TE, low-card features, velocity acceleration, v9 interactions).
- **Топ-30 важливості ознак** за LightGBM gain.
- **Двоступеневу пропозицію рішення** - пороги auto-block / manual review /
  auto-pass, оптимізовані під фіксований ліміт ручної перевірки
  (≤ 10 % користувачів).

## Історія версій

- **v9** (поточна) - додано крос-користувацькі сигнали по картах/власниках,
  LOO toxicity по картах/власниках, target encoding на рівні транзакцій,
  сигнали low-card-користувачів, velocity acceleration. CatBoost замінив
  другий LightGBM для справжньої різноманітності моделей. Early stopping
  збільшено з 200 до 300 раундів.
- **v8** (референс, опційна) - базовий набір ознак без крос-користувацьких
  сигналів; вмикається лише якщо `features_v8.py` присутній поряд з
  `ensemble_v9.py`.
